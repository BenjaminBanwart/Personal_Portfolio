Notable Projects

REST-Rant

REST-Rant is a full stack application that allows users to perform full CRUD operations (create, read, update, delete) to create reviews on anything they would like. The reviews can be manipulated as data and can even hold comments, specific to each review. The technologies that this project utilizes include: Bootstrap, MongoDB, Express, NodeJS, JSX, method-override, dotenv, and express-react-views. The application is hosted and live on Heroku.

MERNstagram

MERNstagram is a PERN-stack application that was created as a milestone project for my course at KSU. It was created in collaboration with three other classmates simulating a dev team environment. This project was very instrumental in showing all of us how to properly function as a team and properly use all of the different version control features of git. The technologies used in development include P:postgreSQL, E:Express, R:React, N:Node.js, ElephantSQL, PgAdmin, Material-UI, and Axios. The application is built as a monorepo in the github repository and is deployed in three locations. The client side, or front-end, is hosted on Netlify, the server, or back-end, is hosted on Heroku, and the SQL database is hosted with ElephantSQL.

Bread-CRUD

Bread-CRUD is a full stack application that functions as a bakery. All of the different data relationships are shown throughout the application where every bread has a baker, and every baker could have baked multiple breads. The user can "order" a bread and choose which baker they would like to bake his or her bread. The technologies that this project utilizes include: Bootstrap, MongoDB, Express, NodeJS, JSX, method-override, dotenv, and express-react-views. The application is hosted and live on Heroku.

Pokemon_Battle_Game

Pokemon_Battle_Game is a simulation of a pokemon battle from the original Pokemon games. In this game, the user is prompted to choose three pokemon that he or she will use to fight against a computer player's (CPU) three randomly selected pokemon. The user fights against the CPU with these three pokemon until either team's three pokemon faint. Whichever team is left standing wins! This game is written in pure JavaScript, CSS, and HTML and runs in the browser. It is hosted and live using Github Pages.

Books-API

Books-API is a JavaScript API that users can send requests to. The database that it interacts with currently stores data and information about books. The database used is a MongoDB database and the application is hosted and live on Heroku. Enter this link into the browser searchbar to send a request to the API:
https://books-api-project-1.herokuapp.com/books . The API returns an array of data back to the user and displays it as JSON in the browser.
